# Project: Slide.html (Interactive AI-Generated html-format Docs Editor)

## 1. Core Vision
Build a hybrid presentation tool where AI handles the heavy lifting (80% structure/design) and the human performs the "last mile" refinements (20% layout/text/style) using a visual, Figma-like interface. The tool ensures a "Clean Code" loop by feeding visual changes back to the AI for semantic refactoring.

---

## 2. Technical Architecture: The "Sandwich" Model
To prevent styling conflicts and ensure a clean editing experience, the app uses a layered approach:

1.  **Content Layer (Iframe/Shadow DOM):** - Renders the raw HTML/CSS generated by the AI.
    - Isolated environment to prevent global CSS leaks.
2.  **Interaction Layer (Overlay):**
    - A transparent UI layer sitting on top of the content.
    - Uses **Moveable.js** to provide handles for dragging, resizing, and rotating.
    - Calculates the `BoundingClientRect` of elements inside the Iframe to position the handles.

---

## 3. The "Round-trip" Engineering Workflow
This is the heart of the project, solving the "dirty code" problem:

1.  **Render:** The app loads a local HTML/CSS file.
2.  **Visual Edit:** The user adjusts elements via mouse (Moveable.js) and edits text (contentEditable).
3.  **Delta Extraction:** Instead of saving the messy inline styles, the app generates a **Visual Delta JSON**.
4.  **AI Refactor (The Sync):** - The [Original Code] + [Visual Delta JSON] are sent to the AI (Gemini/Claude).
    - AI interprets the *intent* (e.g., "User moved this to the right, I should use `justify-end`").
    - AI returns a clean, refactored HTML/CSS file.
5.  **Save:** The local file is overwritten with the high-quality code.

---

## 4. Data Specification: Visual Delta JSON
The data format sent to the LLM to describe changes:

```json
{
  "target_selector": "#slide-element-01",
  "changes": {
    "geometry": {
      "position": { "from": {"x": 20, "y": 20}, "to": {"x": 300, "y": 20} },
      "size": { "from": {"w": 200, "h": 100}, "to": {"w": 250, "h": 120} }
    },
    "style": {
      "background-color": { "from": "#ffffff", "to": "#f0f0f0" },
      "font-size": { "from": "16px", "to": "20px" }
    },
    "content": { "from": "Original Text", "to": "Updated User Text" }
  }
}
```

## 5. Technical Stack & Implementation
- Frontend Framework: Next.js (React) for the editor UI.
- Manipulation Engine: moveable + selecto (for multi-selection).
- Styling Standard: Tailwind CSS (AI's preferred language for layouts).
- Communication: Local File System API (or CLI-based sync) to read/write HTML files.

## 6. AI Agent Prompt Strategy
- When processing the Delta, the Agent must:
- Prioritize Tailwind utility classes over inline styles.
- Infer layout intent (e.g., if multiple items are moved, suggest a flex or grid container).
- Maintain the original design system (colors, spacing) unless explicitly changed.
- Round sub-pixel values (e.g., 10.02px -> 10px) for cleaner code.
